# 最新搜索

用户问任何需要查询/搜索的问题时，自动做到：

1. 搜索最新信息，截止到当前北京时间
2. 覆盖所有平台：GitHub、网页、文档
3. 标注搜索结果的日期，让用户判断时效性

## 触发条件

以下情况自动触发（不需要用户说"搜最新的""截止到今天"）：
- 搜模型、软件、工具
- 搜技术方案、API 文档
- 搜 GitHub 项目
- 任何需要"最新资料"的问题

## 执行流程

当用户问需要搜索的问题时：

### 1. 先获取当前北京时间
```bash
curl -sI "https://www.baidu.com" 2>/dev/null | grep -i "^date:"
```

### 2. 并行搜索多个平台
- **网页搜索**：WebSearch（当前年份的关键词）
- **GitHub 搜索**：WebSearch `site:github.com <关键词>`
- **中文资源**：WebSearch `<中文关键词> 2026`

### 3. 输出带日期的结果
每个搜索结果标注发布日期，帮用户判断新旧。

## 输出格式

```
**搜索时间**：2026年X月X日 北京时间 XX:XX

| 来源 | 名称 | 日期 | 备注 |
|------|------|------|------|
| GitHub | xxx | 2026-05-28 | 最新 |
| 网页 | xxx | 2025-12-01 | 较旧 |
```

## 规则

- 优先展示最近更新的结果
- 标注每个结果的日期，让用户自己判断
- 过时（超过半年）的结果要明确标注"可能已过时"
- 不要等用户说"搜一下最新的"——看到搜索需求就直接按最新标准执行
